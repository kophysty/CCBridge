# Output — READ-ONLY deliverables

Финализированные артефакты для распространения / клиентов.

> **READ-ONLY.** Никогда не редактируется. Новая версия = новый файл.
> Это правило строже R-004 (общее версионирование).

---

## Что сюда попадает

- Зарелизенные демо
- Финализированные диаграммы
- Подписанные документы (если будут)
- PDF / HTML экспорты для внешних потребителей

## Что сюда НЕ попадает

- Drafts документации (это `docs/`)
- Внутренние планы (`Projects/`)
- Исходный код (`src/`)
- Логи / нарратив (`Discovery/logs/`)

---

## Версионирование

```
Output/
├── v0.1.0/
│   └── ...
├── v0.2.0/
│   └── ...
```

Версии **не перезаписываются** даже при минорных правках. Если нужно
переиздать — `v0.1.0-fix1/`.

См. [R-004](../Rulebook/R-004-preference-versioning-files.md).
