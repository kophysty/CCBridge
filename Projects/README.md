# Projects — где что лежит

Документы по фазам, спринтам, версиям, спайкам.

> **Перед чтением:** [`ROADMAP.md`](../ROADMAP.md) — единая точка входа,
> ссылается на все плановые документы.

---

## Структура

```
Projects/
├── 00-strategy/        # Сквозные документы проекта
│                       # (vision, манифест, roadmap-документы,
│                       #  стратегические решения, договоры)
│
├── cross-cutting/      # Сквозные планы, архитектура,
│                       # session-handoffs, gap-анализы,
│                       # task-coverage документы
│
├── v0.1-mvp/           # Конкретная версия / релиз
│                       # (план реализации, scope, доработки в
│                       #  рамках этой версии)
│
└── (vN-feature/)       # Будущие версии
```

---

## Правила размещения

```
  Тип документа                              Куда
  ─────────────────────────────────────────  ──────────────────────────
  Сквозной (весь проект, жизненный цикл)      00-strategy/
   - vision, манифест, roadmap-документы
   - глобальные принципы

  Сквозная архитектура                         cross-cutting/
   - схемы, planning документы по
     нескольким версиям
   - session-handoffs между Claude/Codex

  Материалы конкретной версии                  vN-<feature>/
   - план реализации этой версии
   - scope + acceptance
   - аудиты, дизайны для этой версии
```

**Что НЕ класть в `00-strategy/`:**
- scope конкретной фазы
- план одной версии
- черновики

**Что НЕ класть в `cross-cutting/`:**
- планы конкретных версий

---

## Версионирование документов

Документы существенной правке = новая версия (R-004):
- `<plan>-v1.md` → `<plan>-v2.md` (не перезаписываем)
- В шапке указывать `Заменяет: vN`

См. [R-004](../Rulebook/R-004-preference-versioning-files.md).

---

## Связь с ROADMAP

Каждая папка `vN-*` упоминается в `ROADMAP.md` хотя бы одной записью
(Active / Queued / Pending). Без записи в ROADMAP — план невидим
и теряется. См. [R-007](../Rulebook/R-007-workflow-planning-discipline.md).
