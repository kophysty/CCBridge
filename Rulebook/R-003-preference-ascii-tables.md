# R-003 — ASCII-таблицы в .md с выровненными колонками

**Приоритет:** ⚪ P3
**Категория:** preference
**Статус:** active
**Введено:** 2026-05-02
**Источник:** заимствовано из Oil Automation `R-013`. Pipe-таблицы
без выравнивания становятся нечитаемыми при редактировании.

---

## Контекст

Markdown-таблицы рендерятся одинаково независимо от выравнивания
колонок в исходнике. Но **исходник читают и редактируют**. Невыравненные
pipe-таблицы становятся кашей, в которой невозможно понять структуру
без рендера.

---

## Правило

Все таблицы в `.md` файлах оформляются как **ASCII-таблицы с
выровненными колонками** через пробелы для паддинга, в блоках кода
с тройными бэктиками.

```
  Колонка 1     Колонка 2     Колонка 3
  ────────────  ────────────  ────────────
  значение      значение      значение
  значение2     значение2     значение2
```

**Не использовать** стандартный markdown pipe-format (`|---|---|`)
кроме случаев, когда таблица действительно короткая (≤3 колонки,
≤3 строки) и значения короткие.

---

## Плохой пример

```markdown
| Версия | Срок | Цена | Зависимости |
|--------|------|------|-------------|
| v0.1 | 1 неделя | 0 | базовое |
| v0.2 | 2 недели | 0 | требует деплоя hotfix.5 |
| v0.3 | 3 недели | 0 | требует MCP SDK >= 1.0 |
```

(При редактировании выравнивание ломается, читать невозможно.)

## Хороший пример

````markdown
```
  Версия    Срок         Цена   Зависимости
  ────────  ───────────  ─────  ─────────────────────────────────
  v0.1      1 неделя     0      базовое
  v0.2      2 недели     0      требует деплоя hotfix.5
  v0.3      3 недели     0      требует MCP SDK >= 1.0
```
````
