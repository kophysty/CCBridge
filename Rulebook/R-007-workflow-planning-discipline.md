# R-007 — Planning discipline: plan-файл + ROADMAP в одном коммите

**Приоритет:** 🟡 P1
**Категория:** workflow
**Статус:** active
**Введено:** 2026-05-02
**Источник:** заимствовано из Oil Automation `R-031`. Spike или план,
не записанный в ROADMAP, теряется на 4-5 дней — пока кто-то случайно
не наткнётся на plan-файл.

---

## Контекст

При начале нового спринта / спайка / эксперимента появляется план-файл
в `Projects/cross-cutting/` или `Projects/vN-*/`. Если в `ROADMAP.md`
нет соответствующей записи (Active / Queued / Spike) — этот план:

- Не виден при старте новой сессии (CLAUDE.md велит читать ROADMAP)
- Теряется в подкаталогах
- Дублирующий план может появиться неделей позже от другого agent'а

ROADMAP — **единственная точка входа** «что в работе». Plan-файлы =
детали; без записи в ROADMAP плана как бы не существует.

---

## Правило

**Любой plan-файл = запись в `ROADMAP.md` в том же коммите.**

Запись в ROADMAP содержит:

- **ID/название** — короткое, узнаваемое
- **Статус** — Active / Queued / Pending / Spike
- **Plan-link** — относительный путь до файла
- **Trigger** — что должно произойти, чтобы перейти из Queued в Active

Без этих 4 полей запись не считается валидной.

---

## Плохой пример

Коммит #1:
```
Projects/v0.1-mvp/PR1-core-modules-plan.md   ← создан
ROADMAP.md                                    ← не тронут
```

→ Через 3 дня другая сессия не видит этого плана при чтении ROADMAP,
делает дубль или начинает другую задачу.

## Хороший пример

Коммит #1:
```
Projects/v0.1-mvp/PR1-core-modules-plan.md   ← создан
ROADMAP.md                                    ← добавлена запись:

  v0.1.0-PR1   PR1: core modules            Plan: Projects/v0.1-mvp/
  (Active)      (verdict, lockfile, state,   PR1-core-modules-plan.md
                audit_log, migrations,       Trigger: после ОК
                config)                       аудита (✅ 28.04)
```

---

## Как проверить (защита)

- **CLAUDE.md** — секция «обязательные сверки перед задачей»:
  «План спринта / spike → R-007 (запись в ROADMAP)»
- **AGENTS.md** — раздел «Planning»
- **code** — нет (полагаемся на дисциплину)

---

## Исключения

- Тривиальные TODO в рамках одного коммита — не нужен plan-файл
- Bugfix без отдельного спринта — записывается в `[Unreleased]`
  CHANGELOG, не в ROADMAP
- Эксперимент **в worktree без main**, который точно будет выкинут —
  spike-запись в ROADMAP всё равно нужна (со статусом Spike), чтобы
  потом не повторять
